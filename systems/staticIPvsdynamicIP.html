    Redes: Conceptos Básicos y Configuración

    Introducción: ¿Qué es una IP?

    IP (Internet Protocol): Es un número único que identifica un dispositivo en una red. Funciona como la "dirección" de tu casa, pero en el mundo digital. Gracias a la IP, los dispositivos pueden comunicarse entre sí.

    Existen dos versiones principales:

    • IPv4: Formato clásico, como 192.168.1.1.
    • IPv6: Formato más largo, diseñado para soportar más dispositivos.

    Sin una IP, tu dispositivo no podría enviar ni recibir datos en la red.

    Otros términos importantes

    • DHCP: Protocolo que asigna IPs automáticamente.
    • Gateway: Puerta de enlace para salir de la red local hacia Internet.
    • DNS: Sistema que traduce nombres de dominio en direcciones IP.
    • TCP: Protocolo que garantiza la entrega de datos.
    • UDP: Protocolo rápido, sin confirmación de entrega.

    2. IP de Gateway y DNS

    Gateway: Es la puerta de enlace que conecta tu red local con otras redes, normalmente Internet. El Gateway actúa como intermediario: cuando tu dispositivo quiere acceder a una página web, envía los datos al Gateway, que los redirige hacia Internet.

    Además, el Gateway suele realizar funciones de enrutamiento (decide por dónde enviar los datos) y NAT (Network Address Translation), que permite que varios dispositivos compartan una sola IP pública para salir a Internet.

    Sin un Gateway, tu equipo solo podría comunicarse con dispositivos dentro de la misma red local.

    Ejemplo práctico

    En una red doméstica, el router es el Gateway. Si tu router tiene IP 192.168.1.1, todos los dispositivos envían sus solicitudes a esa dirección para salir a Internet.

    gateway 192.168.1.1
dns-nameservers 8.8.8.8 8.8.4.4

    Diagrama del flujo

    graph LR A[PC del Usuario] --> B[Gateway/Router] B --> C[Proveedor de Internet] C --> D[Servidor DNS] D --> E[Servidor Web]

    DNS: Convierte nombres de dominio (ej. www.google.com) en direcciones IP. Sin DNS, tendrías que escribir la IP del servidor para acceder a una web.

    3. Puertos de comunicación

    Los puertos son números que identifican servicios en un dispositivo. Por ejemplo, el puerto 80 se usa para HTTP y el puerto 22 para SSH.

    Tipos: TCP (conexión fiable) y UDP (rápido, sin confirmación).

    Comandos para ver puertos

    # Con netstat
netstat -tuln

# Con ss
ss -tuln

# Escaneo con nmap desde otra máquina
nmap -p 1-65535 192.168.1.100

    4. Configuración de Netplan

    Netplan es la herramienta en Ubuntu para configurar redes mediante archivos YAML.

    Ejemplo IP estática

    OJO! en este ejemplo la tarjeta se llama eth0, revisad con ip a si se llama igual o, por ejemplo, enp0s3 ;)

    network:
  version: 2
  ethernets:
    eth0:
      dhcp4: no
      addresses: [192.168.1.100/24]
      gateway4: 192.168.1.1
      nameservers:
        addresses: [8.8.8.8, 8.8.4.4]

    Ejemplo IP dinámica

    network:
  version: 2
  ethernets:
    eth0:
      dhcp4: yes
