Enunciado

Un colegio necesita un sistema para gestionar a sus empleados y almacenar la información en una base de datos relacional externa. El sistema debe incluir una interfaz gráfica desarrollada con Java Swing y aplicar herencia e interfaces en el modelado orientado a objetos. La persistencia se realizará mediante JDBC.

1. Clases e interfaz (2 puntos)

  • Defina una clase abstracta Empleado con:
    • nombre: String
    • sueldoBase: double (valor por defecto 1000.0 €)
    • horasTrabajo: int (valor por defecto 8)
  • Defina la interfaz Trabajo con los métodos:
    • trabajar(): String → devuelve la descripción de la actividad.
    • calcularSueldo(): double → devuelve el sueldo total del empleado.

2. Herencia (2 puntos)

  • Profesor: bono preparación clases 500 €. trabajar() → “El profesor está impartiendo clases y preparando material.”
  • Conserje: plus mantenimiento 200 €. trabajar() → “El conserje está realizando tareas de mantenimiento y limpieza.”
  • Director: bonus gestión 1000 €, trabaja 6 horas/día. trabajar() → “El director está supervisando el colegio y organizando el personal.”

3. Clase Colegio (1 punto)

  • Atributos: nombre: String, empleados: List<Empleado>.
  • Métodos:
    • añadirEmpleado(Empleado e)
    • mostrarActividades() → recorre todos los empleados e imprime su trabajar().
    • calcularSueldos() → muestra el sueldo por empleado y el total por tipo y global.

4. Conexión a BBDD (JDBC) (2 puntos)

  • Crear la base de datos colegio_db y la tabla empleados: 
    CREATE TABLE `empleados` (
  id INT AUTO_INCREMENT PRIMARY KEY,
  nombre VARCHAR(50) NOT NULL,
  tipo VARCHAR(20) NOT NULL,   -- 'Profesor', 'Conserje', 'Director'
  sueldo DOUBLE NOT NULL
);
  • Al añadir un empleado en la aplicación, persistir sus datos en la tabla (INSERT).
  • Al iniciar la aplicación, cargar los empleados existentes desde la BBDD (SELECT) y mostrarlos en la interfaz.
  • Utilice PreparedStatement para evitar inyección SQL.

5. Interfaz gráfica (Swing) (2 puntos)

  • Ventana principal con:
    • Campo para el nombre del colegio.
    • Botones: Añadir empleado, Mostrar empleados, Calcular sueldos.
    • Área de texto (o tabla) para listar actividades y sueldos.
  • Formulario modal para añadir empleado (tipo y nombre).
  • Estilo y diseño limpios; manejo de errores y mensajes de validación.

6. Clase Main (1 punto)

  • Inicializa la conexión JDBC a la BBDD.
  • Crea el objeto Colegio y la ventana Swing.
  • Permite interacción completa sin consola (todo vía GUI).
