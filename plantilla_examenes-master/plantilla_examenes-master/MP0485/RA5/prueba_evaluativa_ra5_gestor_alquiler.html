Enunciado

Desarrolla una aplicación Java que simule la gestión interna de una empresa de alquiler vacacional (similar a Airbnb o Booking). El sistema permitirá gestionar las viviendas disponibles y las reservas de clientes, utilizando una arquitectura MVC y un entorno gráfico basado en Swing.

La información se almacenará en dos ficheros de texto, simulando una base de datos interna del sistema.

3. Funcionalidades de la Aplicación (Swing) (5 puntos) (las pantallas tienen que funcionar, no se puntúa si solo se ha hecho el diseño)

  • Diseño gráfico: Ventanas creadas con JFrame y cuadros de diálogo JOptionPane.
  • Ver viviendas disponibles: Mostrar únicamente las viviendas que no estén asociadas a ninguna reserva. [1 punto]
  • Listar alojamientos actuales: Mostrar los clientes alojados y la vivienda ocupada por cada uno. [1 punto]
  • Crear reserva: Introducir los datos del cliente, mostrar las viviendas disponibles y asignar una vivienda al cliente. [2 puntos]
  • Borrar reserva: Eliminar una reserva existente a partir del DNI del cliente. [1 punto]

4. Persistencia de Datos (2 puntos)

  • Fichero de viviendas: Se carga al iniciar la aplicación y contiene todas las viviendas disponibles.
  • Fichero de reservas: Guarda las reservas activas (clientes y viviendas asignadas).

5. Validaciones y Restricciones (1 punto)

  • Una vivienda solo puede estar asignada a un cliente.
  • Un cliente no puede tener más de una reserva.
  • Tratamiento de errores: Mostrar un diálogo asociado a una excepción personalizada en caso de intentar crear una reserva con un cliente que ya tiene una reserva o asignar una vivienda que ya está ocupada.

Si no compila o se presenta con errores de código, la nota será 0.
