Enunciado

Un banco necesita una aplicación para gestionar sus tarjetas. Existen diferentes tipos de tarjetas, con características y límites distintos, que afectan la operativa de la entidad financiera.

El programa se ejecutará por consola y deberá hacer uso de herencia e interfaces para modelar correctamente el comportamiento de las tarjetas.

1. Clase base Tarjeta (1 punto)

  • Atributos comunes a todas las tarjetas:
    • Número de tarjeta
    • Titular
    • Saldo inicial: 0€

2. Interfaz Operacion (1 punto)

  • Defina la interfaz Operacion con los siguientes métodos:
    • pagar(monto): realiza un pago y actualiza el saldo.
    • consultarSaldo(): devuelve el saldo actual de la tarjeta.

3. Clase Banco (1 punto)

  • Atributos:
    • Nombre del banco
    • Listado de tarjetas
  • Métodos:
    • Añadir tarjetas al banco.
    • Mostrar información y saldo de todas las tarjetas.

4. Herencia de tarjetas (2 puntos)

  • TarjetaDebito
    • Permite pagos hasta el saldo disponible.
    • consultarSaldo(): devuelve el saldo disponible.
    • pagar(monto): “Se ha descontado X€ de la tarjeta de débito.”
  • TarjetaCredito
    • Límite de crédito: 2000€
    • consultarSaldo(): muestra saldo y crédito disponible.
    • pagar(monto): “Se ha cargado X€ a la tarjeta de crédito.”

5. Clase Main y ejecución del programa (5 puntos)

  • Crear un banco solicitando su nombre al usuario.
  • Permitir crear tarjetas mediante un menú:
    • Elegir tipo de tarjeta (débito o crédito).
    • Introducir titular y número de tarjeta.
    • Crear dos tarjetas de cada tipo.
    • Añadir automáticamente las tarjetas al banco.
  • Realizar pagos de prueba y mostrar los resultados.
  • Mostrar el saldo final de cada tarjeta.
