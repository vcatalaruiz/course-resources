Enunciado

Una empresa de logística desea una aplicación para gestionar los vehículos de los que dispone. Cada tipo de vehículo presenta comportamientos distintos según su medio de transporte.

El programa se ejecutará por consola y deberá hacer uso de herencia e interfaces para modelar correctamente el comportamiento de los vehículos.

1. Clase base Vehículo (2 puntos)

  • Propiedades comunes a todos los vehículos:
    • Licencia
    • Combustible
    • Disponible
  • Método:
    • cargar(): imprime por consola “cargando”.

2. Interfaz Navegable (1 punto)

  • Defina la interfaz Navegable con el método:
    • navegar()

3. Interfaz Aéreo (1 punto)

  • Defina la interfaz Aereo con el método:
    • volar()

4. Subclase Barco (2 puntos)

  • Propiedad adicional:
    • Puerto donde está amarrado
  • Implementa navegar():
    • Imprime “navegando”
    • Deja de estar disponible
    • El puerto se borra
    • Reduce el combustible en -10
  • Sobrescribe cargar():
    • Imprime “cargando en puerto ” + puerto

5. Subclase Avión (2 puntos)

  • Propiedad adicional:
    • Aeropuerto de despegue/aterrizaje
  • Implementa volar():
    • Imprime “volando”
    • Deja de estar disponible
    • El aeropuerto se borra
    • Reduce el combustible en -30
  • Sobrescribe cargar():
    • Imprime “cargando en aeropuerto ” + aeropuerto

6. Clase Main (2 puntos)

  • Crear un barco con:
    • Licencia: 111
    • Combustible: 50
    • Puerto: Valencia
    • Disponible
  • Crear un avión con:
    • Licencia: 222
    • Combustible: 200
    • Aeropuerto: Palma
    • Disponible
  • Cargar y navegar el barco.
  • Cargar y volar el avión.
  • Mostrar todos los datos finales de ambos vehículos.
