Enunciado

Una cadena de hoteles necesita una aplicación para gestionar sus habitaciones, clientes y reservas. El programa será utilizado por empleados de recepción y funcionará íntegramente por consola.

El sistema permitirá administrar reservas, asignar habitaciones, registrar servicios adicionales y realizar distintas consultas mediante un menú interactivo.

1. Modelado de clases (3 puntos)

  • Cree una clase por cada elemento identificable del sistema.
  • Cada clase debe disponer de un constructor.
  • Añada getters y setters a todas las clases.
  • Sobrescriba toString() cuando sea necesario.

2. Clases del sistema (8 clases)

  • Hotel
    • Nombre
    • Dirección
    • Listado de habitaciones
    • Listado de reservas
  • Empleado
    • Nombre
    • Apellido
    • Puesto (recepción, limpieza, dirección)
  • Cliente
    • Nombre
    • DNI
    • Teléfono
  • Habitación
    • Número
    • Tipo (individual, doble, suite)
    • Precio por noche
    • Disponible u ocupada
  • Reserva
    • Cliente
    • Habitación
    • Fecha de entrada
    • Fecha de salida
    • Importe total
  • Servicio
    • Nombre del servicio
    • Precio
  • Factura
    • Reserva asociada
    • Listado de servicios
    • Total a pagar
  • Fecha
    • Día
    • Mes
    • Año

3. Clase HotelApp y arranque del programa (2 puntos)

  • El programa debe iniciarse desde una clase HotelApp.
  • Al comenzar:
    • Solicitar los datos del empleado.
    • Crear el objeto empleado.
    • Mostrar el mensaje:
      “Bienvenido/a NOMBRE APELLIDO”

4. Menú de gestión hotelera (5 puntos)

  • Dar de alta cliente.
  • Dar de alta habitación.
  • Crear reserva.
  • Añadir servicios a una reserva.
  • Mostrar factura de una reserva.
  • Cancelar reserva.
  • Salir del programa.
