Enunciado

Una compañía de suministro eléctrico necesita una aplicación para gestionar los contratos de luz de sus clientes. El programa será utilizado por empleados del departamento de atención al cliente y funcionará íntegramente por consola.

El sistema permitirá dar de alta clientes, crear contratos eléctricos, registrar consumos y generar facturas mediante un menú interactivo.

1. Modelado de clases (3 puntos)

  • Cree una clase por cada elemento identificable del sistema.
  • Cada clase debe disponer de un constructor.
  • Añada getters y setters a todas las clases.
  • Sobrescriba toString() cuando sea necesario.

2. Clases del sistema (8 clases)

  • CompañíaElectrica
    • Nombre
    • Listado de contratos
    • Listado de clientes
  • Empleado
    • Nombre
    • Apellido
    • Departamento
  • Cliente
    • Nombre
    • DNI
    • Dirección
  • ContratoLuz
    • Número de contrato
    • Cliente asociado
    • Tarifa
    • Potencia contratada
    • Fecha de inicio
    • Activo o cancelado
  • Tarifa
    • Nombre de la tarifa
    • Precio por kWh
    • Cuota fija mensual
  • Consumo
    • Mes
    • Año
    • kWh consumidos
  • Factura
    • Contrato asociado
    • Consumo facturado
    • Importe total
  • Fecha
    • Día
    • Mes
    • Año

3. Clase LuzApp y arranque del programa (2 puntos)

  • El programa debe iniciarse desde una clase LuzApp.
  • Al comenzar:
    • Solicitar los datos del empleado.
    • Crear el objeto empleado.
    • Mostrar el mensaje:
      “Bienvenido/a NOMBRE APELLIDO”

4. Menú de gestión de contratos (5 puntos)

  • Dar de alta cliente.
  • Crear contrato de luz.
  • Registrar consumo mensual.
  • Generar factura.
  • Mostrar datos de un contrato.
  • Cancelar contrato.
  • Salir del programa.
