Enunciado

Reconoce y utiliza herramientas de desarrollo para crear, probar, optimizar y documentar aplicaciones mediante código y diagramas.

1. Creación del repositorio (1 punto)

  • Crea un repositorio en GitHub llamado MP0487_EXAMEN.
  • (Requerido para evaluación) Configúralo como privado.
  • (Requerido para evaluación) Invita al profesor al repositorio.

2. Primer commit (1 punto)

  • Clonar el repositorio examen proporcionado por el profesor.
  • Haz un cambio y commit de mensaje "first commit" y push a tu repositorio.

3. Gestión de Issues (1 punto)

  • Copia el Issue del repositorio examen.
  • Asígnate el Issue a ti mismo.

4. Trabajo con ramas (1 punto)

  • Crea una nueva rama (branch) desde GitHub.

5. Commits y Pull Request (4 puntos)

  • Cambia a esa rama en tu entorno local.
  • Realiza las modificaciones necesarias para resolver el Issue.
  • Haz commit con el mensaje: "final commit #1".
  • Haz push a la rama de Github.
  • Crea un Pull Request hacia la rama principal.
  • Aprueba el PR y realiza el merge.

6. Evidencias del proceso (2 puntos)

  • (Requerido para evaluación) Adjunta en el Issue capturas de pantalla del proyecto funcionando y aquello que demuestre que el issue ha sido resuelto.
  • Finalmente, cierra el Issue.

Recordatorio del flujo Git profesional

  • 1️⃣ Crear Issue → describir qué se hará.
  • 2️⃣ Crear rama → asociada al Issue.
  • 3️⃣ Commit(s) → con mensajes claros.
  • 4️⃣ Push → subir la rama remota.
  • 5️⃣ Pull Request → revisión del cambio.
  • 6️⃣ Merge → integrar en main.
  • 7️⃣ Cerrar Issue → añadir evidencia.
