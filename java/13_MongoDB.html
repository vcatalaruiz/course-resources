    Acceso a datos con MongoDB

    1. Introducción a MongoDB

    MongoDB es una base de datos NoSQL orientada a documentos que almacena la información en formato BSON (Binary JSON). Está diseñada para ofrecer flexibilidad, escalabilidad y alto rendimiento.

    1.1. Problemas de las bases de datos relacionales

    • Escalabilidad: escalar verticalmente tiene límites físicos.
    • Flexibilidad: cualquier cambio requiere modificar el esquema.
    • Rendimiento: uso intensivo de joins.

    3. Modelo basado en documentos

    En MongoDB los datos se almacenan como documentos JSON dentro de colecciones. No es obligatorio que todos los documentos tengan la misma estructura. 

    
{
  "_id": "64fa12",
  "nombre": "Gran Hotel",
  "direccion": "Calle Mayor 3",
  "habitaciones": 120
}

    4. Relaciones en MongoDB

    MongoDB no usa joins tradicionales. Las relaciones se modelan de dos formas:

    • Embedding (Documentos embebidos)
    • Referencing (Referencias)

    4.1. Relaciones embebidas (Embedding)

    Se utilizan cuando los datos se consultan juntos con frecuencia (relación 1:N). 

    
{
  "_id": 1,
  "hotel": "Gran Hotel",
  "habitaciones": [
    { "numero": 101, "precio": 80 },
    { "numero": 102, "precio": 90 }
  ]
}

    Ventajas

    • Menos consultas
    • Mejor rendimiento

    Desventajas

    • Duplicación de datos
    • Límite de 16MB por documento

    4.2. Relaciones por referencia (Referencing)

    
{
  "_id": 101,
  "numero": 101,
  "hotel_id": 1
}

    Se recomienda cuando los datos crecen mucho o se reutilizan en múltiples colecciones.

    5. CRUD con MongoDB

    MongoDB se puede usar desde Java mediante el MongoDB Java Driver.

    5.1. Conexión

    
MongoClient client = MongoClients.create("mongodb://localhost:27017");
MongoDatabase db = client.getDatabase("empresa");
MongoCollection col = db.getCollection("hoteles");

    5.2. Create (Insertar)

    
Document hotel = new Document("nombre", "Gran Hotel")
        .append("direccion", "Calle Mayor 3")
        .append("habitaciones", 120);

col.insertOne(hotel);

    5.3. Read (Consultar)

    
for (Document d : col.find()) {
    System.out.println(d.toJson());
}

    5.4. Update (Actualizar)

    
col.updateOne(
    Filters.eq("nombre", "Gran Hotel"),
    Updates.set("direccion", "Avenida Central 10")
);

    5.5. Delete (Eliminar)

    
col.deleteOne(Filters.eq("nombre", "Gran Hotel"));
