RA 4 - Programación Orientada a Objetos

Introducción, Constructores y Modificadores de Acceso

1. Introducción a la Programación Orientada a Objetos

La Programación Orientada a Objetos (POO) es un paradigma que permite modelar entidades del mundo real mediante clases y objetos. Cada objeto combina datos (atributos) y comportamientos (métodos).

Hasta el momento, para poder identificar un objeto teníamos que declarar variables independientes. Esto no es muy mantenible y puede generar errores.

En el mundo real, la representación de una entidad es la suma de sus propiedades. Es lo que nos hace diferenciar, por ejemplo, un hombre de un vehículo, un animal de una planta, etc

1.1. ¿Qué es una clase y un objeto?

Una clase es el molde que define cómo serán los objetos. Un objeto es una instancia concreta de esa clase.

1.2. Ejemplo básico


// Definición de una clase básica en Java
public class Vehiculo {
    String marca;       // Atributo que indica la marca del vehículo
    String modelo;      // Atributo que indica el modelo
    int anyo;            // Atributo que indica el anyo de fabricación

    // Método que muestra información del vehículo
    public void mostrarInfo() {
        System.out.println("Marca: " + marca);
        System.out.println("Modelo: " + modelo);
        System.out.println("Año: " + anyo);
    }
}

// Crear un objeto e inicializar sus atributos
public class Main {
    public static void main(String[] args) {
        Vehiculo coche = new Vehiculo(); // Instancia de la clase Vehiculo
        coche.marca = "Toyota";          // Asignación de valores
        coche.modelo = "Corolla";
        coche.anyo = 2020;
        coche.mostrarInfo();             // Llamada al método
    }
}

2. Constructores

Un constructor es un método especial que se ejecuta automáticamente al crear un objeto. Sirve para inicializar sus atributos. Si no se define, Java proporciona uno por defecto.

2.1. Características

  • Tiene el mismo nombre que la clase.
  • No devuelve ningún valor (ni siquiera void).
  • Puede estar sobrecargado (varios constructores con diferentes parámetros).

2.2. Ejemplo con sobrecarga


// Clase con constructores sobrecargados
public class Hotel {
    String nombre;
    String direccion;

    // Constructor por defecto
    public Hotel() {
        this.nombre = "Sin nombre"; // Valor inicial por defecto
        this.direccion = "Desconocida";
    }

    // Constructor con parámetros
    public Hotel(String nombre, String direccion) {
        this.nombre = nombre;       // Inicializa el atributo nombre
        this.direccion = direccion; // Inicializa el atributo dirección
    }

    public void mostrarInfo() {
        System.out.println("Hotel: " + nombre + ", Dirección: " + direccion);
    }
}

// Uso en el método main
public class Main {
    public static void main(String[] args) {
        Hotel h1 = new Hotel(); // Usa constructor por defecto
        Hotel h2 = new Hotel("Gran Hotel", "Calle Mayor 3"); // Usa constructor con parámetros
        h1.mostrarInfo();
        h2.mostrarInfo();
    }
}

3. Encapsulamiento

El encapsulamiento consiste en ocultar los detalles internos de una clase y exponer solo lo necesario. Para ello se usan modificadores de acceso y métodos getters y setters.

3.1. ¿Por qué es necesario el encapsulamiento?

Imagina que estás construyendo un coche. ¿Dejarías que cualquier persona toque el motor sin control? Probablemente no, porque podría romper algo importante. Lo mismo ocurre en programación: si todos los atributos y métodos son públicos, cualquiera puede modificarlos sin seguir las reglas que tú has definido, y eso puede provocar errores graves.

El encapsulamiento consiste en proteger la lógica interna de una clase. No se trata de ocultar información porque sí, sino de garantizar que los cambios se hagan de forma controlada.

Por ejemplo, si tienes un atributo saldo en una cuenta bancaria, no quieres que alguien lo cambie directamente a un valor negativo. En su lugar, ofreces métodos como ingresar() o retirar() que validan las operaciones.

Ejemplo práctico: Cuenta bancaria con encapsulamiento


// Clase que representa una cuenta bancaria
public class CuentaBancaria {
    private double saldo; // Atributo privado para proteger el estado

    // Constructor para inicializar el saldo
    public CuentaBancaria(double saldoInicial) {
        if (saldoInicial >= 0) {
            this.saldo = saldoInicial;
        } else {
            this.saldo = 0; // Si el saldo inicial es negativo, lo dejamos en 0
        }
    }

    // Método para ingresar dinero
    public void ingresar(double cantidad) {
        if (cantidad > 0) {
            saldo += cantidad; // Suma la cantidad al saldo
            System.out.println("Ingreso realizado: " + cantidad);
        } else {
            System.out.println("Error: la cantidad debe ser positiva.");
        }
    }

    // Método para retirar dinero
    public void retirar(double cantidad) {
        if (cantidad > 0 && cantidad <= saldo) {
            saldo -= cantidad; // Resta la cantidad del saldo
            System.out.println("Retiro realizado: " + cantidad);
        } else {
            System.out.println("Error: cantidad inválida o saldo insuficiente.");
        }
    }

    // Getter para consultar el saldo actual
    public double getSaldo() {
        return saldo;
    }
}

// Uso en el método main
public class Main {
    public static void main(String[] args) {
        CuentaBancaria cuenta = new CuentaBancaria(100); // Saldo inicial de 100
        cuenta.ingresar(50); // Ingresamos 50
        cuenta.retirar(30);  // Retiramos 30
        cuenta.retirar(200); // Intento inválido (saldo insuficiente)
        System.out.println("Saldo actual: " + cuenta.getSaldo());
    }
}

Aunque tú seas el programador, piensa que tu código puede crecer, trabajar con otros módulos o incluso ser usado por otros desarrolladores. Si todo es público, cualquiera puede alterar el estado del objeto sin respetar las reglas que lo hacen funcionar correctamente. El encapsulamiento es como poner puertas y llaves: no impide usar la casa, pero sí evita que entren por la ventana.

3.2. Modificadores de acceso

Una buena práctica consiste en añadir modificadores de acceso a las variables de la clase. Su objetivo es mantener la consistencia de la lógica del propio objeto. Por ejemplo, imaginemos un objeto que representa un vehículo y todos sus parámetros son accesibles desde cualquier punto; el programador podría utilizar sin control los atributos y cambiar la "esencia" del objeto.

Por otro lado, para proteger la lógica, el programador no tiene por qué conocer todos los atributos y métodos de la clase.. ¿Conoces todas las herramientas que has utilizado? Por ejemplo, Scanner o System.out.println(..). Una forma de mantener la lógica de la clase es hacer accesibles únicamente aquellos métodos o atributos que realmente se tienen que utilizar.

  • public: Accesible desde cualquier clase.
  • private: Solo accesible dentro de la clase.
  • protected: Accesible desde la clase, subclases y el mismo paquete.
  • sin modificador Accesible desde cualquier clase dentro de su mismo paquete.

3.3. getters y setters

Una buena práctica consiste en acceder a los atributos de la clase mediante métodos, evitando su manipulación directa. Esto permite tener un mayor control sobre la lógica e información del propio objeto.

3.4. Ejemplo getters y setters


// Clase con atributos privados y métodos públicos para acceder a ellos
public class Persona {
    private String nombre; // Atributo privado
    private int edad;      // Atributo privado

    // Constructor
    public Persona(String nombre, int edad) {
        this.nombre = nombre;
        this.edad = edad;
    }

    // Getter para nombre
    public String getNombre() {
        return this.nombre;
    }

    // Setter para nombre
    public void setNombre(String nombre) {
        this.nombre = nombre;
    }

    // Getter para edad
    public int getEdad() {
        return this.edad;
    }

    // Setter para edad
    public void setEdad(int edad) {
        this.edad = edad;
    }
}

// Uso en el método main
public class Main {
    public static void main(String[] args) {
        Persona p = new Persona("Ana", 25);
        System.out.println("Nombre: " + p.getNombre());
        p.setEdad(26); // Cambia la edad usando el setter
        System.out.println("Edad actualizada: " + p.getEdad());
    }
}

4. Especificadores STATIC y FINAL – STATIC

El modificador static indica que un atributo o método pertenece a la clase y no a una instancia concreta. Esto significa que el valor o comportamiento es compartido por todos los objetos creados a partir de esa clase.

Además, podemos combinar static con métodos públicos y privados para controlar el acceso y la lógica interna. Los métodos públicos son accesibles desde cualquier parte, mientras que los privados solo se pueden usar dentro de la clase.

4.1. Características del modificador static

  • Los atributos static son compartidos por todas las instancias de la clase.
  • Los métodos static se pueden llamar sin crear un objeto, usando el nombre de la clase.
  • No se puede acceder a atributos no estáticos desde un método estático.

4.2. Ejemplo práctico con métodos públicos, privados y estáticos


// Clase que demuestra el uso de métodos públicos, privados y estáticos
public class Hotel {
    private String nombre;           // Atributo privado
    private String direccion;        // Atributo privado
    public static String categoria = "Estrellas"; // Atributo estático compartido

    // Constructor público
    public Hotel(String nombre, String direccion) {
        this.nombre = nombre;
        this.direccion = direccion;
    }

    // Método público para mostrar información del hotel
    public void mostrarInfo() {
        System.out.println("Hotel: " + nombre + ", Dirección: " + direccion);
        mostrarMensajePrivado(); // Llamada a método privado
    }

    // Método privado para lógica interna
    private void mostrarMensajePrivado() {
        System.out.println("Este mensaje es interno y no accesible fuera de la clase.");
    }

    // Método estático público para obtener la categoría
    public static String getCategoria() {
        return categoria;
    }

    // Método estático privado para lógica interna
    private static void mensajeInternoStatic() {
        System.out.println("Método estático privado ejecutado.");
    }

    // Método público que llama al método estático privado
    public static void ejecutarInterno() {
        mensajeInternoStatic();
    }
}

// Uso en el método main
public class Main {
    public static void main(String[] args) {
        Hotel h1 = new Hotel("Gran Hotel", "Calle Mayor 3");
        Hotel h2 = new Hotel("Hotelito", "Avenida Central");

        h1.mostrarInfo(); // Llama a método público y privado interno
        h2.mostrarInfo();

        // Acceso al atributo y método estático sin crear objeto
        System.out.println("Categoría general: " + Hotel.getCategoria());

        // Ejecutar método estático que llama a otro privado
        Hotel.ejecutarInterno();
    }
}

Significado de los símbolos

Símbolo Significado
+ Método o atributo público (accesible desde cualquier clase).
- Método o atributo privado (solo accesible dentro de la clase).
# Método o atributo protegido (accesible desde la clase y sus subclases).
static Indica que el elemento pertenece a la clase y no a una instancia concreta.
: Tipo Muestra el tipo de dato del atributo o el valor devuelto por el método.
