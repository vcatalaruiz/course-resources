    Acceso a datos con OODB

    1. Introducción a ObjectDB

    ObjectDB es una base de datos orientada a objetos (OODB) que almacena información directamente como objetos Java. Está diseñada para ofrecer alta velocidad, persistencia transparente y compatibilidad con JPA.

    1.1. Ventajas sobre bases de datos relacionales

    • Persistencia de objetos: no es necesario mapear manualmente a tablas.
    • Transparencia JPA: las operaciones CRUD se integran con Java de forma natural.
    • Alto rendimiento: consultas optimizadas sobre objetos.

    2. SQL vs OODB

    2.1. Datos

    Mientras que SQL requiere normalización y tablas, OODB almacena objetos completos, respetando la estructura de la aplicación.

    2.2. Conceptos clave

    • Entidades (Entity)
    • Atributos de objeto
    • Relaciones entre objetos (1:1, 1:N, N:M)

    2.3. Uso

    OODB es ideal para aplicaciones Java, APIs REST, juegos, simuladores y sistemas donde los objetos son complejos y la integridad de relaciones es importante.

    3. Modelo basado en objetos

    En ObjectDB, los datos se almacenan como objetos Java persistentes. Cada entidad se representa mediante una clase con anotaciones JPA. 

    
@Entity
public class Hotel {
    @Id @GeneratedValue
    private long id;
    private String nombre;
    private String direccion;
    private int habitaciones;
}

    4. Relaciones entre objetos

    Las relaciones en OODB se representan mediante referencias directas entre objetos:

    • 1:1, 1:N y N:M mediante atributos de objeto y listas
    • Embebido o Referencia según necesidad de acceso o compartición

    4.1. Ejemplo embebido (1:N)

    Modelo conceptual OODB

    classDiagram class Hotel { +id +nombre +direccion } class Habitacion { +numero +precio } Hotel "1" o-- "*" Habitacion : embebido 
    
@Entity
public class Hotel {
    @Id @GeneratedValue
    private long id;
    private String nombre;
    @OneToMany(cascade=CascadeType.ALL)
    private List habitaciones;
}

@Entity
public class Habitacion {
    private int numero;
    private double precio;
}

    Ventajas

    • Acceso directo a los objetos relacionados
    • Consultas más rápidas

    Desventajas

    • Puede duplicar información si se comparte

    5. CRUD con ObjectDB y Java

    ObjectDB se maneja mediante JPA y EntityManager en Java.

    5.1. Conexión y creación de EntityManager

    
EntityManagerFactory emf = Persistence.createEntityManagerFactory("objects/p2.odb");
EntityManager em = emf.createEntityManager();

    5.2. Create (Insertar)

    
em.getTransaction().begin();
Point p = new Point(1, 2);
em.persist(p);
em.getTransaction().commit();

    5.3. Read (Consultar)

    
TypedQuery query = em.createQuery("SELECT p FROM Point p", Point.class);
List results = query.getResultList();
for(Point p : results) {
    System.out.println(p);
}

    5.4. Update (Actualizar)

    
em.getTransaction().begin();
Query update = em.createQuery("UPDATE Point p SET p.x = p.x + 100 WHERE p.x < :threshold");
update.setParameter("threshold", 5);
update.executeUpdate();
em.getTransaction().commit();
em.clear(); // Importante para refrescar

    5.5. Delete (Eliminar)

    
em.getTransaction().begin();
Query delete = em.createQuery("DELETE FROM Point p WHERE p.x >= :minX");
delete.setParameter("minX", 103);
delete.executeUpdate();
em.getTransaction().commit();
em.clear(); // Refrescar

    6. Puesta a punto (setup)

    6.1. Instalación

    Descarga ObjectDB desde https://www.objectdb.com/download/2.9.2 y descomprime el paquete.

    6.2. Dependencia

    Añade la dependencia de ObjectDB en tu proyecto. 

    
<!-- Define ObjectDB repository -->
<repositories>
    <repository>
        <id>objectdb</id>
        <name>ObjectDB Repository</name>
        <url>https://m2.objectdb.com</url>
    </repository>
</repositories>
<dependencies>
    <!-- ObjectDB dependency -->
    <dependency>
        <groupId>com.objectdb</groupId>
        <artifactId>objectdb</artifactId>
        <version>2.9.2</version>
    </dependency>
</dependencies>

    6.3. Ejecutar

    En la carpeta bin de la instalación, ejecuta: 

    java -jar explorer-2.9.2.jar

    Este comando abre el visualizador de ObjectDB (ObjectDB Explorer) para inspeccionar archivos .odb.

    Resumen clave

    • ObjectDB es OODB orientada a objetos
    • Se integra con JPA y Java
    • Almacena objetos persistentes sin esquema rígido
    • Relaciones directas entre objetos (1:1, 1:N, N:M)
    • CRUD sencillo y eficiente con EntityManager
