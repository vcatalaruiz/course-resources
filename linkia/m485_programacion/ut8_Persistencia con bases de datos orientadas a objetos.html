Introducción a la persistencia en POO

Bases de datos relacionales vs orientadas a objetos

Las bases de datos relacionales almacenan información en tablas y requieren un mapeo entre objetos y registros. Las bases de datos orientadas a objetos almacenan directamente los objetos del programa, eliminando la necesidad de ORM.

  • RDB: tablas, SQL, claves foráneas.
  • OODB: objetos, referencias directas, consultas orientadas a objetos.
  • Mejor rendimiento en jerarquías y relaciones complejas.

Ventajas de las OODB

Las OODB se integran de forma natural con lenguajes orientados a objetos, manteniendo la coherencia entre el modelo de dominio y la persistencia.

  • Soporte nativo para herencia y polimorfismo.
  • Persistencia de objetos complejos y anidados.
  • Reducción de errores al no usar ORM.

Características de una OODB

Una base de datos orientada a objetos almacena instancias de clases respetando los principios de la POO.

  • Persistencia de objetos.
  • Identidad única mediante OID.
  • Encapsulamiento de datos.
  • Soporte completo para herencia.
  • Navegación por referencias entre objetos.

Instalación y configuración de ObjectDB

ObjectDB es un gestor de bases de datos orientadas a objetos para Java. Es fácil de instalar y configurar, y se integra con JPA.

  • Descarga el JAR desde el sitio oficial: https://www.objectdb.com/download
  • Añade el JAR al classpath de tu proyecto Java:
    1. En NetBeans: clic derecho en el proyecto → Propiedades → Bibliotecas → Añadir JAR/Carpeta.
    2. En Eclipse: clic derecho en el proyecto → Build Path → Add External Archives.
    3. En IntelliJ: File → Project Structure → Libraries → + → Java y selecciona el JAR.
  • Configura la unidad de persistencia en persistence.xml.

Modelo de objetos persistentes

El esquema de una OODB se define mediante clases. Cada objeto almacenado es una instancia persistente de una clase del modelo. 

public class Usuario {
    private String nombre;
    private String email;
    private int edad;

    public Usuario(String nombre, String email, int edad) {
        this.nombre = nombre;
        this.email = email;
        this.edad = edad;
    }
}

Sistemas gestores OODBMS

Existen distintos gestores de bases de datos orientadas a objetos, tanto comerciales como de código abierto. En el entorno formativo se utiliza principalmente ObjectDB por su integración con JPA.

  • ObjectDB (Java, compatible con JPA).
  • ZODB (Python).
  • GemStone y Versant (entornos empresariales).

Instalación y uso de ObjectDB

ObjectDB es un gestor ligero que no requiere instalación compleja. Basta con añadir su JAR al proyecto Java y configurar la unidad de persistencia.

  • JDK 8 o superior.
  • IDE Java (NetBeans, Eclipse, IntelliJ).
  • Archivo objectdb.jar en el classpath.

Conexión a ObjectDB desde Java

Mediante JPA se establece la conexión y se persisten objetos usando EntityManager. 

EntityManagerFactory emf =
    Persistence.createEntityManagerFactory("usuarioPU");
EntityManager em = emf.createEntityManager();

Usuario u = new Usuario("Ana", "ana@email.com", 29);
em.getTransaction().begin();
em.persist(u);
em.getTransaction().commit();

em.close();
emf.close();

Diseño de clases persistentes

Las clases persistentes definen la estructura de almacenamiento en una OODB. Se anotan con JPA y actúan como esquema de la base de datos. 

@Entity
public class Producto {
    @Id @GeneratedValue
    private long id;
    private String nombre;
    private double precio;

    public Producto() {}
}

Operaciones CRUD

JPA permite crear, leer, actualizar y eliminar objetos persistentes de forma sencilla mediante transacciones.

  • Crear: persist()
  • Leer: find() y consultas JPQL
  • Actualizar: modificar atributos dentro de transacción
  • Eliminar: remove()

Consultas con JPQL

JPQL es un lenguaje de consultas orientado a objetos que permite recuperar información usando entidades y sus relaciones. 

TypedQuery<Producto> query =
    em.createQuery(
        "SELECT p FROM Producto p WHERE p.precio > 500",
        Producto.class);
List<Producto> resultados = query.getResultList();

Gestión de datos estructurados y compuestos

Las OODB permiten almacenar relaciones directas entre objetos, como listas y jerarquías de clases, respetando el modelo orientado a objetos.

  • Relaciones uno a muchos con @OneToMany.
  • Persistencia de objetos compuestos.
  • Soporte para herencia y consultas polimórficas.

Resumen de la unidad

En esta unidad se ha aprendido a gestionar la persistencia de datos respetando el paradigma orientado a objetos, utilizando OODB y JPA para diseñar, almacenar y consultar objetos de forma eficaz.

  • Persistencia de objetos completos.
  • Uso de ObjectDB y JPA en Java.
  • Operaciones CRUD y consultas JPQL.
  • Gestión de relaciones y jerarquías de clases.

La persistencia de datos permite que la información generada por una aplicación se conserve más allá de su ejecución. En programación orientada a objetos, esto implica almacenar objetos completos, incluyendo su estructura, atributos y relaciones, en sistemas externos como bases de datos.

  • Los datos sobreviven al cierre del programa.
  • Se almacenan objetos, no solo valores.
  • Se mantiene la estructura del modelo de clases.
