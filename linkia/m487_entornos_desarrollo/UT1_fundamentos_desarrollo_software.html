1. El sistema informático y su relación con el software

Un sistema informático está formado por componentes físicos (hardware) y programas (software) que trabajan de forma conjunta para procesar datos y ejecutar aplicaciones.

La coordinación entre ambos determina la estabilidad, rendimiento y funcionalidad del sistema:

  • Hardware: CPU, memoria, almacenamiento, periféricos.
  • Software: sistemas operativos, aplicaciones, utilidades.

1.1 Componentes del sistema: memoria, CPU y periféricos

Los tres pilares fundamentales del sistema informático son la memoria, la CPU y los periféricos. Trabajan en conjunto para ejecutar programas y permitir la interacción con el usuario y el entorno.

  • Memoria: almacena datos e instrucciones temporal o permanentemente.
  • CPU: ejecuta instrucciones y coordina la actividad interna.
  • Periféricos: permiten la interacción con el exterior. Pueden ser de entrada, salida o mixtos.

1.1.1 Memoria

La memoria es indispensable para la ejecución eficiente de programas. Su función principal es almacenar datos e instrucciones que la CPU necesita para procesar información.

Se clasifica en:

  • Memoria volátil (RAM): rápida y temporal; almacena procesos activos.
  • Memoria no volátil: HDD, SSD, flash; almacena datos permanentes.

El rendimiento global del sistema depende en gran medida de la capacidad y velocidad de la RAM.

1.1.2 CPU (Unidad Central de Procesamiento)

Es el “cerebro” del sistema. Ejecuta instrucciones y coordina el funcionamiento general. Es la parte más crítica para el rendimiento del sistema, ya que determina la velocidad de procesamiento y la capacidad de multitarea.

Se compone de tres partes principales:

  • Unidad de Control (CU): gestiona el flujo de instrucciones.
  • ALU: ejecuta operaciones aritméticas y lógicas.
  • Registros: almacenamiento interno ultrarrápido.

Sus características clave son: frecuencia (GHz) y número de núcleos.

1.1.3 Periféricos

Permiten la interacción entre el sistema informático y el exterior. Su función es facilitar la entrada y salida de información.

Existen tres tipos principales de periféricos:

  • Entrada: teclado, ratón, escáner, micrófono.
  • Salida: pantalla, impresora, altavoces.
  • Mixtos: webcam, auriculares, mando de juego.

1.2 Interacción entre hardware y software

El software da instrucciones y el hardware ejecuta acciones. Cuando un programa se ejecuta, el sistema operativo traduce las instrucciones del software en comandos que el hardware puede entender y ejecutar.

El sistema operativo controla el flujo de información entre ambos:

  • Gestiona dispositivos y recursos.
  • Coordina CPU y memoria.
  • Atiende peticiones de aplicaciones.

1.2.1 Relación entre componentes físicos y lógicos

El software (lógico) solicita servicios que el hardware (físico) ejecuta. Esta relación se canaliza a través del sistema operativo y las APIs.

  • Abstracciones: archivos, procesos, hilos, sockets.
  • Interfaces: llamadas al sistema, bibliotecas y controladores.

Ejemplos de abstracciones: archivos, procesos, hilos, sockets.

Ejemplos de interfaces: llamadas al sistema, bibliotecas y controladores.

El software no interactúa directamente con el hardware, sino a través de capas de abstracción que facilitan la portabilidad y seguridad.

1.2.2 Coordinación por el sistema operativo

El SO planifica CPU, gestiona memoria, y orquesta E/S de periféricos, resolviendo concurrencia y protegiendo recursos.

  • Planificación de CPU (quantum, cambio de contexto).
  • Gestión de memoria (segmentación/paginación, memoria virtual).
  • E/S e interrupciones; cola de dispositivos.

1.2.3 Papel del sistema operativo y controladores

El SO ofrece servicios y seguridad; los drivers traducen las órdenes del SO al lenguaje del dispositivo.

  • Servicios del SO: archivos, procesos, red, seguridad.
  • Controladores: impresora, GPU, cámara, NIC.

2. Tipos de software y clasificación funcional

El software engloba todo el conjunto de programas que permiten realizar tareas. Se clasifica según su propósito y función.

Existen tres tipos principales de software:

  • Software de sistema: gestiona el hardware y permite ejecutar aplicaciones.
  • Software de aplicación: realiza tareas específicas solicitadas por el usuario.
  • Software de programación: herramientas para desarrollar otros programas.

2.1 Software de sistema

  • Sistema operativo: Windows, Linux, macOS, Android.
  • Controladores: impresoras, GPU, audio, red.

2.2 Software de aplicación

  • Productividad: Word/Docs, Excel/Sheets, Presentaciones.
  • Específicas: ERP/CRM, CAD, edición multimedia.

2.3 Software de programación

  • Lenguajes: Java, Python, C#, C++.
  • IDEs: VS Code, Visual Studio, IntelliJ, Eclipse.

2.4 Software de utilidades

  • Seguridad: antivirus, antimalware.
  • Mantenimiento: desfragmentación, limpieza de disco.
  • Copias de seguridad y recuperación.

2.5 Clasificación según licencia

  • Propietario: uso bajo licencia; código cerrado.
  • Software libre: libertades de uso, estudio, modificación y distribución.
  • Open‑source: código disponible con condiciones de la licencia.

Ejemplos: Windows (propietario), GNU/Linux (libre), Firefox (open‑source).

3. Fases del ciclo de vida del software

El ciclo de vida del software (SDLC) es un conjunto de fases que siguen los proyectos de desarrollo desde su inicio hasta su despliegue y mantenimiento.

Las etapas típicas del SDLC son:

  1. Planificación y análisis: definición de objetivos, requisitos y viabilidad del proyecto.
  2. Diseño e Implementación: creación de la arquitectura, diseño detallado y escritura del código fuente.
  3. Pruebas: verificación de la funcionalidad, corrección de errores y validación del software.
  4. Despliegue: lanzamiento del software en el entorno de producción.
  5. Mantenimiento: actualización, mejora y corrección de errores post-despliegue.

3.1 Modelos del ciclo de vida

Modelo en cascada

  • Pros: simple, fases claras, documentación fuerte.
  • Contras: poca flexibilidad ante cambios, feedback tardío.
  • Uso: entornos muy regulados o requisitos estables.

3.2 Modelo ágil (iterativo e incremental)

  • Pros: adaptación al cambio, entregas frecuentes, feedback continuo.
  • Contras: requiere disciplina, difícil escalar sin prácticas sólidas.
  • Uso: producto evolutivo, requisitos cambiantes.

3.3 Modelo de prototipos

  • Pros: valida requisitos temprano, mejora usabilidad.
  • Contras: expectativas irreales si se confunde prototipo con producto.
  • Uso: requisitos poco definidos al inicio.

4. Código fuente, objeto y ejecutable

En el proceso de desarrollo de software, el código pasa por varias etapas:

  1. Código fuente: escrito por el programador en un lenguaje de alto nivel. Ejemplo: Java, C++, Python.
  2. Código objeto: resultado de compilar el código fuente, aún no ejecutable. Ejemplo: archivos .o o .obj.
  3. Código ejecutable: archivo final listo para ejecutar tras enlazar con bibliotecas. Ejemplo: archivos .exe o .out.

5. Máquinas virtuales y código intermedio

Las máquinas virtuales y el código intermedio son conceptos clave en el desarrollo de software moderno, que permiten portabilidad, seguridad y optimización en la ejecución de programas.

  • Máquinas virtuales: software que emula un sistema informático completo, permitiendo ejecutar programas en un entorno aislado.
  • Código intermedio: formato de código que se encuentra entre el código fuente y el ejecutable final, diseñado para ser ejecutado por una máquina virtual.

5.1 Ventajas y desventajas (VM y código intermedio)

  • Ventajas: portabilidad, aislamiento/seguridad, optimización JIT.
  • Desventajas: sobrecarga en tiempo de ejecución, dependencia de la VM.

5.2 Ejemplos prácticos

  • Java: fuente → .class (bytecode) → JVM.
  • .NET: fuente C# → IL → CLR (JIT a nativo).
  • Python: .py → bytecode .pyc → PVM.

6. Clasificación de lenguajes de programación

Los lenguajes de programación se clasifican según tres criterios principales:

Nivel de abstracción:

  • Bajo nivel (Ensamblador), cerca del hardware.
  • Medio nivel (C), equilibrio entre control y facilidad.
  • Alto nivel (Java, Python, C++), más fácil y portable.

Paradigmas:

  • Imperativo: describe cómo se debe hacer algo (C, Python).
  • Orientado a Objetos: organiza el código en objetos con atributos y métodos (Java, C++).
  • Funcional: se basa en funciones puras sin estado (Haskell, Lisp).
  • Lógico: se basa en reglas y hechos para resolver problemas (Prolog).

Forma de ejecución:

  • Compilados: generan ejecutables optimizados (C, C++, Rust).
  • Interpretados: se ejecutan línea a línea (Python, JavaScript).

6.1 Ejemplos y usos típicos

  • C: sistemas embebidos, SO, control de hardware.
  • Java: backend empresarial, Android (Kotlin/Java).
  • Python: ciencia de datos, scripting, automatización.
  • JavaScript: front-end web, Node.js en servidor.
  • Haskell: programación funcional pura, investigación.
  • Prolog: lógica, IA simbólica, rule‑based.

7. Herramientas de desarrollo

Las herramientas de desarrollo son esenciales para facilitar el proceso de creación, prueba y mantenimiento del software.

  • IDEs: editor, depurador, compilador.
  • Control de versiones: Git, SVN.
  • Herramientas de depuración: puntos de ruptura, inspección de variables.
  • Pruebas automatizadas: unitarias, integración, regresión.

Las herramientas se evaluan según su facilidad de uso, compatibilidad con el ecosistema, rendimiento y soporte de la comunidad.

7.1 Control de versiones (Git)

  • Historial de cambios, ramas (branching) y fusiones (merging).
  • Trabajo colaborativo: repos remotos (GitHub, GitLab).

7.2 Depuración

  • Breakpoints, inspección de variables, ejecución paso a paso.
  • Watch/trace, evaluación de expresiones, call stack.

7.3 Pruebas automatizadas e integración continua

  • Pruebas: unitarias, integración, regresión.
  • CI: builds y tests automáticos (Jenkins, GitHub Actions).

7.4 Criterios para evaluar herramientas

  • Facilidad de uso y curva de aprendizaje.
  • Compatibilidad, rendimiento y ecosistema de plugins.
  • Soporte y comunidad.

Ejemplos: Visual Studio, Eclipse/IntelliJ, VS Code, Jenkins.
