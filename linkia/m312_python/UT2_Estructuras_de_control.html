Estructuras de selección

1.1 Condicional simple if

La estructura if permite ejecutar un bloque de código cuando una condición booleana es verdadera. Se usa para tomar decisiones simples sin alternativa explícita.

Descripción

Se evalúa la condición y si es verdadera se ejecuta el bloque indentado; si no, se continúa con el flujo.

Ejemplo 
numero = int(input('Introduce un número: '))
if numero % 2 == 0:
    print('Bienvenido, has introducido un número par')

1.2 Condicional doble if-else

if-else ofrece una alternativa cuando la condición es falsa. Siempre se ejecuta exactamente uno de los dos bloques.

1.3 Condicional múltiple if-elif-else

Cuando hay más de dos opciones, elif permite evaluar condiciones adicionales de forma ordenada. El primer if/elif verdadero ejecuta su bloque y se ignoran los siguientes.

Estructuras de repetición

2.1 Bucle while

while repite un bloque mientras la condición sea verdadera. Es útil cuando no se conoce de antemano el número de iteraciones. Hay que cuidar que la condición cambie en algún momento para evitar bucles infinitos.

Ejemplo 
contador = 0
while contador < 5:
    print('contador =', contador)
    contador += 1

2.2 Bucle for y función range()

for itera sobre secuencias (listas, cadenas, ranges). range() genera secuencias numéricas eficientemente.

Ejemplo 
for i in range(5):           # 0 1 2 3 4
    print('i =', i)

# recorrer lista
nombres = ['Ana', 'Luis', 'Marta']
for n in nombres:
    print(n)

2.3 Bucle anidado

Un bucle anidado es un bucle dentro de otro. Se usa para recorrer estructuras bidimensionales o para algoritmos que combinan iteraciones. Tener en cuenta la complejidad temporal que aumenta multiplicativamente.

Ejemplo 
for i in range(3):
    for j in range(2):
        print('i =', i, 'j =', j)

Sentencias de salto (break, continue)

Estas sentencias alteran el flujo dentro de bucles y bloques:

  • break interrumpe el bucle más cercano inmediatamente.
  • continue salta a la siguiente iteración del bucle, omitiendo el resto del bloque actual.
Ejemplo 
for n in range(10):
    if n == 5:
        break          # sale del for
    if n % 2 == 0:
        continue       # omite imprimir pares
    print(n)

3.2 Control del flujo de ejecución

El control de flujo combina condicionales, bucles y sentencias de salto para modelar la lógica del programa. Diseñar claramente las condiciones y evitar bucles infinitos o rutas lógicas inaccesibles es clave para programas correctos y mantenibles.

Control de errores

4.1 Estructura try-except

try permite ejecutar código que puede generar excepciones y atraparlas con bloques except para evitar que el programa termine abruptamente. Es fundamental validar entradas externas o realizar operaciones inseguras.

Ejemplo 
try:
    x = int(input('Introduce un entero: '))
    print('Has escrito', x)
except ValueError:
    print('No has introducido un entero válido')

4.2 Captura de errores comunes

Algunos errores frecuentes y su manejo:

  • ValueError al convertir cadenas a números: usar try-except ValueError.
  • ZeroDivisionError al dividir por cero: comprobar divisor o capturar la excepción.
Ejemplo 
try:
    a = int(input('a: '))
    b = int(input('b: '))
    result = a // b
except ZeroDivisionError:
    print('Error: división por cero')
except ValueError:
    print('Error: entrada no entera')
else:
    print('Resultado', result)

4.3 Cláusulas finally y excepciones personalizadas

finally se ejecuta siempre, ocurra o no una excepción, útil para liberar recursos. También podemos definir excepciones personalizadas heredando de Exception para casos específicos de la aplicación.

Ejemplo 
class MiError(Exception):
    pass

try:
    raise MiError('Algo específico ha fallado')
except MiError as e:
    print('MiError capturado:', e)
finally:
    print('Cierre de recursos o limpieza')

Prueba y depuración

5.1 Mensajes de error y advertencias

Mensajes claros ayudan a identificar la causa del fallo. Usar mensajes que indiquen la entrada esperada y la acción a tomar facilita la corrección por parte del usuario. Las advertencias (por ejemplo con el módulo warnings) sirven para alertar sin detener la ejecución.

Ejemplo 
import warnings

def dividir(a, b):
    if b == 0:
        warnings.warn('División por cero, se devolverá None', UserWarning)
        return None
    return a / b

5.2 Técnicas de depuración manual

Técnicas útiles:

  • Insertar print() para inspeccionar valores intermedios.
  • Verificar supuestos y precondiciones con aserciones assert.
  • Reducir el código a un ejemplo mínimo que reproduzca el fallo.
Ejemplo de aserción 
def suma_lista(lst):
    assert isinstance(lst, list), 'lst debe ser una lista'
    return sum(lst)

5.3 Uso del depurador en el IDE (Thonny, VSCode)

Los IDEs ofrecen herramientas visuales para depurar:

  • Puntos de interrupción (breakpoints) para detener la ejecución en una línea.
  • Inspección de variables en tiempo de ejecución.
  • Ejecutar paso a paso (step into, step over, step out) para entender el flujo.
