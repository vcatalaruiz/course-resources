1. Entrada/salida por consola

La consola es la forma más sencilla y directa de comunicación entre el usuario y el programa en los primeros niveles de aprendizaje. Mediante la entrada se reciben datos desde el teclado, y con la salida se muestran resultados o mensajes por pantalla utilizando principalmente input() y print().

  • La E/S por consola facilita comprender el flujo de datos: el usuario introduce información, el programa la procesa y devuelve resultados.
  • Es la base para introducir validación de datos y formateo de salidas.
  • Sirve como paso previo al trabajo con archivos y formatos más complejos.

1. Entrada/salida por consola

1.1 Interacción con el usuario: input()

La función input() permite pausar la ejecución del programa para que el usuario escriba un dato. El valor leído se almacena siempre como cadena de caracteres (str), por lo que suele ser necesaria una conversión de tipo antes de operar con él.

  • Sintaxis básica: variable = input("Mensaje al usuario: ").
  • El contenido introducido es siempre str, incluso si parece un número.
  • Conversión típica: int() para enteros, float() para decimales, bool() para booleanos (con tratamiento especial).
nombre = input("Introduce tu nombre: ")

edad_texto = input("Introduce tu edad: ")

# Conversión de tipos

edad = int(edad_texto)

print("Hola,", nombre)
print("El año que viene tendrás", edad + 1, "años")

1. Entrada/salida por consola

1.2 Formateo de salidas personalizadas: print()

La función print() muestra información por pantalla. Un uso cuidadoso del formateo es clave para generar salidas claras, alineadas y legibles en programas que presentan informes, cálculos o resultados.

  • Concatenación con +: solo une cadenas, obliga a convertir números con str().
  • Separación con comas: permite mezclar tipos sin conversión explícita; print() inserta espacios automáticamente.
  • f-strings (Python ≥ 3.6): permiten incrustar variables dentro de la cadena y formatear números con precisión.
  • Parámetros útiles: sep (separador) y end (final de línea).
nombre = "Ana"

nota = 8.75

# a) Concatenación

print("Alumno: " + nombre + " - Nota: " + str(nota))

# b) Comas

print("Alumno:", nombre, "Nota:", nota)

# c) f-strings con formateo

print(f"Alumno: {nombre} - Nota: {nota:.2f}")  # 2 decimales

# Personalizar separador y final

print("A", "B", "C", sep="-", end=">")
print("Fin")

2. Manejo de archivos

El manejo de archivos en Python permite almacenar y recuperar datos de forma permanente, más allá de la ejecución del programa. Es una competencia básica en contextos donde se necesitan registros, informes, logs de actividad o cualquier información reutilizable.

  • La función open() permite acceder a archivos de texto mediante distintos modos de apertura.
  • La lectura y escritura de ficheros de texto constituye la base para trabajar con formatos más complejos como CSV o JSON.
  • Se recomienda usar with open() para garantizar el cierre seguro del archivo.

2.1 Lectura y escritura en archivos de texto

Para acceder a un archivo se utiliza la función integrada open(), que recibe el nombre del archivo y el modo de apertura. Una vez abierto, se pueden leer o escribir líneas de texto antes de cerrar el fichero.

  • Primer parámetro: nombre del archivo (por ejemplo, "datos.txt").
  • Segundo parámetro: modo ("r", "w", "a", "r+", etc.).
  • La lectura básica se realiza con read(), readline() o readlines().
# Lectura básica

archivo = open("texto.txt", "r", encoding="utf-8")
contenido = archivo.read()
print(contenido)
archivo.close()

# Escritura básica (sobrescribe el archivo si ya existe)

archivo = open("salida.txt", "w", encoding="utf-8")
archivo.write("Primera línean")
archivo.write("Segunda línean")
archivo.close()

2.2 Modos de apertura (r, w, a, r+)

Los modos de apertura definen qué se puede hacer con el archivo y qué ocurre con su contenido previo. Elegir el modo correcto es crucial para evitar perder información.

  • "r": solo lectura. Produce error si el archivo no existe.
  • "w": escritura. Crea el archivo o borra todo su contenido si ya existe.
  • "a": añadir (append). Escribe al final sin borrar lo anterior.
  • "r+": lectura y escritura. Requiere que el archivo exista.
# Ejemplo de modo append (a)

with open("log.txt", "a", encoding="utf-8") as f:
f.write("Nueva entrada de logn")

2.3 Uso de with open() (gestión automática)

La estructura with open() as es la forma recomendada de trabajar con archivos en Python, ya que garantiza el cierre automático incluso si se produce un error durante la lectura o escritura.

  • No es necesario llamar a .close() explícitamente.
  • Reduce la posibilidad de dejar archivos abiertos accidentalmente.
  • Es más legible y seguro en programas reales.
# Lectura con with

with open("datos.txt", "r", encoding="utf-8") as f:
    for linea in f:
        print(linea.strip())

# Escritura con with

with open("resultado.txt", "w", encoding="utf-8") as f:
    f.write("Informe generado correctamente")

2.4 Estructura de archivos y líneas

Un archivo de texto está formado por líneas separadas por caracteres de fin de línea (generalmente n). Python ofrece distintas funciones para trabajar con estas líneas de forma flexible.

  • read(): lee todo el contenido como una sola cadena.
  • readline(): lee una única línea por llamada.
  • readlines() o iterar sobre el archivo: permite procesar línea a línea.
  • Es frecuente usar .strip() para eliminar saltos de línea al procesar texto.
# Leer línea a línea

with open("alumnos.txt", "r", encoding="utf-8") as f:
    for linea in f:
        print("Línea:", linea.strip())

# Escribir varias líneas

lineas = ["Ana,20n", "Luis,22n", "María,19n"]
with open("alumnos.txt", "w", encoding="utf-8") as f:
    f.writelines(lineas)

3. Organización de datos en ficheros

En entornos profesionales, trabajar con archivos implica no solo leer y escribir texto, sino organizar los datos de forma estructurada. El uso de separadores, formatos y codificaciones permite simular registros, tablas, informes o logs de actividad fácilmente procesables.

  • Un archivo puede verse como una colección de registros (líneas) y campos (datos dentro de cada línea).
  • Los separadores (comas, tabuladores, etc.) delimitan los campos de información.
  • Una codificación adecuada (como utf-8) asegura que se representen correctamente acentos y símbolos especiales.

3.1 Separadores, formatos y codificaciones

Un separador es un símbolo que se usa para delimitar campos en una misma línea de texto. Los formatos como CSV o TSV se basan en separadores y son muy comunes para intercambiar datos tabulados entre programas.

  • Separadores típicos: coma (,) en CSV, tabulador (t) en TSV, punto y coma (;), barra vertical (|).
  • Formatos frecuentes: .txt, .csv, .tsv, .log, .dat.
  • Codificaciones: utf-8 (recomendada), utf-16, latin-1 según el contexto.
# Escritura de un CSV simple

with open("alumnos.csv", "w", encoding="utf-8") as f:
    f.write("nombre,edadn")
    f.write("Ana,20n")
    f.write("Luis,22n")

# Lectura y separación por comas

with open("alumnos.csv", "r", encoding="utf-8") as f:
    next(f)  # saltar cabecera
    for linea in f:
        nombre, edad = linea.strip().split(",")
        print(f"Alumno: {nombre} - Edad: {edad}")

3.2 Aplicaciones prácticas: registros, logs, informes

La organización de datos en ficheros permite crear estructuras sencillas como registros, logs e informes, que son muy utilizadas en aplicaciones reales para llevar trazabilidad y generar resultados legibles.

  • Registros: cada línea representa una unidad de información (usuario, reserva, visita...).
  • Logs: almacenan trazas de funcionamiento, errores, avisos y eventos con marcas de tiempo.
  • Informes: resumen de datos procesados, alineados en columnas para facilitar su lectura.

# Ejemplo de log con marcas de tiempo
import datetime
with open("app.log", "a", encoding="utf-8") as f:
    f.write(f"{datetime.datetime.now()}: Usuario inició sesiónn")
