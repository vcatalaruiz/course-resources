UT3. Funciones, modularidad y tipos compuestos

En esta unidad aprenderás a organizar el código en funciones y a utilizar estructuras de datos compuestas (listas, diccionarios y cadenas) para escribir programas en Python más limpios, modulares y reutilizables.

  • Definición y uso de funciones en Python.
  • Recursividad: funciones que se llaman a sí mismas.
  • Listas, diccionarios y cadenas de texto avanzadas.

1.1 Definición y llamada a funciones

Una función es un bloque de código reutilizable que realiza una tarea específica. Se define con la palabra clave def, un nombre, paréntesis y dos puntos; el cuerpo debe ir correctamente indentado.

  • Mejora la legibilidad y mantenimiento del código.
  • Permite reutilizar lógica en distintos puntos del programa.
  • Se invoca escribiendo su nombre seguido de paréntesis.
def saludar():
    print("Hola, mundo")

# Llamada a la función
saludar()

1.2 Parámetros y retorno de valores

Las funciones pueden recibir información mediante parámetros y devolver resultados con la instrucción return. También es posible definir valores por defecto para algunos parámetros.

  • Los parámetros permiten personalizar el comportamiento de la función.
  • return finaliza la función y devuelve un valor.
  • Los parámetros con valor por defecto se indican como param=valor.
def sumar(a, b=0):
    """Devuelve la suma de a y b."""
    return a + b

print(sumar(3, 5))   # 8
print(sumar(10))     # 10, usa el valor por defecto de b

1.3 Variables locales y globales

Las variables locales solo existen dentro de la función donde se declaran; las variables globales se definen fuera de cualquier función y son accesibles desde distintos puntos del programa.

  • Una variable local deja de existir al terminar la función.
  • Una variable global puede leerse desde funciones.
  • Para modificar una global dentro de una función se usa la palabra clave global.
mensaje = "Hola desde fuera"
def mostrar():
    global mensaje
    mensaje = "Hola desde la función"
    print(mensaje)
mostrar()
print(mensaje)

1.4 Paso por valor y por referencia

En Python los argumentos se pasan por asignación de referencia. Pueden ser:

  • Objetos mutables (listas, diccionarios) pueden modificarse dentro de la función.
  • Objetos inmutables (enteros, cadenas) no cambian su valor original.
  • Es importante conocer el tipo de dato para evitar efectos secundarios inesperados.
def agregar_elemento(lista):
    lista.append(4)  # Modifica la lista original
def incrementar(numero):
    numero += 1      # No modifica el entero original
    return numero
datos = [1, 2, 3]
agregar_elemento(datos)
print(datos)   # [1, 2, 3, 4]
n = 10
print(incrementar(n))  # 11
print(n)               # 10

2. Recursividad

2.1 Concepto y ejemplos básicos

La recursividad es una técnica en la que una función se llama a sí misma para resolver un problema.

  • Cada llamada recursiva trabaja con una versión más simple del problema.
  • Debe existir siempre un caso base que detenga la recursión.
  • Es especialmente útil en problemas definidos de forma inductiva (factoriales, recorridos, etc.). 
def factorial(n):
    if n == 0:            # Caso base
        return 1
    else:                 # Llamada recursiva
        return n * factorial(n - 1)

print(factorial(5))  # 120

2.2 Casos base y llamadas recursivas

El caso base determina cuándo se detiene la recursión. La llamada recursiva debe modificar los parámetros para acercar el problema a ese caso base y evitar bucles infinitos.

  • Sin caso base, la recursión produce un desbordamiento de pila (stack overflow).
  • El caso base debe alcanzarse tras un número finito de llamadas.
  • La llamada recursiva reduce el problema en cada paso.
def suma_recursiva(n):
    if n == 1:           # Caso base
        return 1
    else:                # Llamada recursiva
        return n + suma_recursiva(n - 1)

print(suma_recursiva(5))  # 15

3. Listas

3.1 Creación, recorrido y métodos básicos

Una lista permite almacenar múltiples elementos en una única variable. Es ordenada y mutable, por lo que se pueden añadir, modificar y eliminar elementos.

  • Se define entre corchetes [] separados por comas.
  • Se recorre fácilmente con un bucle for.
  • Dispone de métodos como append(), remove(), sort(), etc. 
frutas = ["manzana", "plátano", "naranja"]
for fruta in frutas:
    print(fruta)
frutas.append("pera")
frutas.sort()
print(frutas)

3.3 Comprensión de listas

La comprensión de listas permite crear listas de forma compacta y legible a partir de iteraciones y condiciones, en una sola línea de código.

  • Sintaxis general: [expresion for elemento in iterable if condicion].
  • Facilita transformaciones y filtrados sobre colecciones.
  • Hace el código más conciso que los bucles tradicionales en muchos casos.
numeros = [1, 2, 3, 4, 5]

cuadrados = [n ** 2 for n in numeros]
pares = [n for n in numeros if n % 2 == 0]

print(cuadrados)  # [1, 4, 9, 16, 25]
print(pares)      # [2, 4]

4. Diccionarios

4.1 Estructura clave–valor y métodos

Un diccionario almacena pares clave–valor. Cada clave es única y sirve para acceder rápidamente a su valor asociado. Es una estructura mutable y muy eficiente para búsquedas.

  • Se definen con llaves {} y pares clave: valor.
  • Acceso directo mediante dic["clave"] o dic.get("clave").
  • Métodos útiles: keys(), values(), items(), pop(), update(). 
persona = {"nombre": "Ana", "edad": 30, "activo": True}

print(persona.get("nombre", "Sin nombre"))

for clave, valor in persona.items():
    print(clave, "=", valor)

4.4 Anidamiento y usos prácticos

Los diccionarios pueden contener otros diccionarios, listas u otras estructuras complejas, lo que permite representar información estructurada como la de varios estudiantes o registros.

  • Permiten modelar datos similares a estructuras JSON.
  • Facilitan agrupar información relacionada bajo una misma clave principal.
  • Son muy útiles en programas que gestionan datos estructurados.
estudiantes = {
    "001": {"nombre": "Ana", "nota": 8.5},
    "002": {"nombre": "Luis", "nota": 7.0}
}

print(estudiantes["001"]["nombre"])  # Ana

5. Cadenas de texto avanzadas

5.1 Métodos de transformación y búsqueda

Las cadenas de texto (strings) son secuencias de caracteres inmutables. Python ofrece numerosos métodos para transformarlas y buscar información dentro de ellas.

  • Transformación: lower(), upper(), capitalize(), title(), strip(), replace().
  • Búsqueda: find(), count(), startswith(), endswith().
  • Permiten normalizar texto, limpiar espacios y localizar subcadenas.
texto = "  Programación en Python  "

print(texto.strip().upper())
print(texto.find("Python"))
print(texto.count("o"))

5.2 Segmentación y formateo de cadenas

Las cadenas permiten realizar las siguientes operaciones:

  • Segmentación: texto[i], texto[inicio:fin].
  • División y unión: split() y join().
  • Formateo: "{}".format(valor) y f"{valor}".
nombre_completo = "Ana García López"
partes = nombre_completo.split()
apellido = partes[-1]
mensaje1 = "Hola, {} {}".format(partes[0], apellido)
mensaje2 = f"Hola, {partes[0]} {apellido}"
print(apellido, mensaje1, mensaje2)
