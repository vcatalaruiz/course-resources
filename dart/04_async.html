Programación Asíncrona en Dart

1. Introducción a la Asincronía

Por defecto, las aplicaciones y programas utilizan un único hilo de ejecución (single-threaded). Cuando se ejecuta un software bajo este modelo secuencial, cualquier operación que requiera un procesamiento denso o una espera externa (como un cálculo complejo o aguardar la respuesta de un servidor) provocará que el programa entero quede completamente bloqueado hasta que dicha acción finalice.

Para solucionar este inconveniente, se recurre a los procesos asíncronos, los cuales impiden la congelación de la interfaz o del flujo de control delegando la tarea en segundo plano. Cabe destacar que no se debe confundir el concepto de multihilo con el de procesos asíncronos.

2. La Clase Future y sus Estados

En Dart, la herramienta fundamental para gestionar operaciones síncronas que postergan su resolución en el tiempo es la clase Future. Mediante un Future, podemos "esperar" de forma segura la obtención de resultados futuros sin bloquear el hilo principal.

Un objeto de tipo Future transiciona obligatoriamente a través de los siguientes estados:

  • Uncompleted (Incompleto): Es el estado inicial que se mantiene de manera persistente mientras la operación asíncrona se encuentra ejecutándose activamente en segundo plano, a la espera de una respuesta.
  • Completed (Completado): Cuando la operación finaliza de manera definitiva, el estado muta a completado. Esta resolución puede concluir de dos formas diferenciadas:
    • Success (Éxito): El proceso ha concluido correctamente y retorna el valor esperado.
    • Error: El proceso ha fallado, devolviendo una excepción o fallo controlado.

2.1. Consumo mediante Callbacks (.then)

El mecanismo clásico para capturar e interactuar con el valor exitoso de un Future una vez es completado consiste en usar el método callback then:


void main(List<String> arguments) {
  print('Inicio del programa.');

  futuro().then((valor) {
    print(valor);
  });

  print('El programa continua...');
}

Future<String> futuro() {
  return Future.delayed(Duration(seconds: 2), () => "¡Esto es el futuro!");
}

3. Palabras Clave: async y await

A fin de simplificar la sintaxis y evitar los anidamientos de callbacks de .then(), Dart introduce el paradigma estructurado de async y await. Esto permite escribir código asíncrono con una apariencia y legibilidad idénticas a las del código síncrono secuencial:

  • async: Palabra clave obligatoria que se añade inmediatamente antes del cuerpo de cualquier función que requiera ejecutar tareas asíncronas internamente.
  • await: Instrucción que suspende la ejecución lineal de la función actual de manera temporal hasta que el Future correspondiente resuelva su valor, sin congelar el resto del software.

void main(List<String> arguments) async {
  print('Inicio del programa.');

  // El flujo espera aquí la resolución antes de pasar a la siguiente línea
  String respuesta = await futuro();
  print(respuesta);

  print('El programa continua...');
}

Future<String> futuro() {
  return Future.delayed(Duration(seconds: 2), () => "¡Esto es el futuro!");
}

4. Tratamiento de Errores Asíncronas

Dado que los procesos asíncronos interactúan con factores impredecibles del entorno (caídas de red, fallos de infraestructura, parsing incorrecto), estos pueden derivar con frecuencia en errores o excepciones. Controlar estas anomalías es vital para evitar el cierre inesperado de la aplicación.

4.1. Captura con .catchError

Si se utiliza la sintaxis tradicional basada en métodos encadenados, se acopla la función catchError al final de la estructura:


void main(List<String> arguments) {
  futuro()
      .then((value) => print('res: $value'))
      .catchError((error) => print('error: $error'));

  print('El programa continua...');
}

Future futuro() async {
  var resultado;
  await Future.delayed(Duration(seconds: 2), () {
    resultado = getServerInfo("https://server.ip/get_info");
  });
  return resultado;
}

5. Peticiones HTTP Externas (Consumo de JSON)

Una de las necesidades más recurrentes en el desarrollo de software actual es la consulta de APIs web externas que proveen datos estructurados en formato JSON. Al depender estrictamente de internet, estas operaciones son forzosamente asíncronas y deben ser orquestadas mediante Future y bloques estructurados de control de excepciones try-catch-finally: 


import 'package:http/http.dart' as http;
import 'package:http/retry.dart';

Future<void> main(List<String> arguments) async {
  final client = RetryClient(http.Client());
  
  try {
    // Realiza la lectura asíncrona de la API REST externa
    print(await client.read(Uri.parse('https://jsonplaceholder.typicode.com/comments')));
  } catch (error) {
    print('Se ha producido un error en la comunicación: $error');
  } finally {
    // Garantiza de manera estricta el cierre de las conexiones del cliente HTTP
    client.close();
  }
}

Resumen de Programación Asíncrona

  • Las tareas asíncronas previenen la congelación y el bloqueo del único hilo principal ante cálculos masivos o peticiones de red.
  • La clase Future encapsula un valor diferido, pasando de un estado uncompleted a un fin completed (éxito o error).
  • La dupla de modificadores async y await dota al entorno asíncrono de una lectura lineal y una semántica limpia de mantener.
  • Es obligatorio controlar los fallos potenciales del flujo asíncrono aplicando cláusulas como catchError o bloques try-catch.
  • El consumo de endpoints e interconexiones HTTP externas requiere siempre el uso de estructuras asíncronas combinadas con un cierre seguro de clientes en bloques finally.
