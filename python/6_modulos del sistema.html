    Módulos del sistema

    1. Introducción: módulos del sistema en Python

    En este tema trabajaremos con módulos de la biblioteca estándar que permiten interactuar con el sistema operativo, crear herramientas de línea de comandos, ejecutar procesos externos y comunicarnos por red.

    • Objetivo: comprender y aplicar os, sys, subprocess, socket y argparse en scripts profesionales.
    • Casos de uso: automatización, utilidades de administración, clientes/servidores básicos y CLIs reproducibles.
    • Buenas prácticas: portabilidad, manejo de errores, mensajes claros y seguridad.
    Dominar estos módulos multiplica la capacidad de tus scripts para integrarse con el sistema y otros programas.

    2. Módulo os: trabajar con archivos y carpetas

    El módulo os proporciona utilidades para interactuar con el sistema operativo: gestionar rutas y directorios, crear/eliminar/mover archivos y carpetas, consultar información del sistema y leer variables de entorno. A continuación se detallan las operaciones más habituales con ejemplos comentados.

    2.1. ¿En qué carpeta estoy? ¿Cómo construyo rutas portables?

    • os.getcwd(): devuelve la carpeta actual (CWD).
    • os.path.join(...): une trozos de rutas sin importar el sistema operativo.
    # EJEMPLO: mostrar la carpeta actual y construir la ruta a un PDF dentro de "docs"
import os

# Obtener el "current working directory" (carpeta desde donde corre el script)
cwd = os.getcwd()
print("Carpeta actual:", cwd)

# Unir partes de una ruta de forma segura (evita escribir "/" o "\" a mano)
ruta_pdf = os.path.join("docs", "manuales", "instalacion.pdf")
print("Ejemplo de ruta:", ruta_pdf)  # Se adapta a Windows/Linux/macOS

    Ejercicio

    Solución

    2.2. Comprobar si existe (archivo o carpeta)

    • os.path.exists(ruta): ¿existe?
    • os.path.isdir(ruta) / os.path.isfile(ruta): ¿es carpeta o archivo?
    # EJEMPLO: comprobar un nombre concreto y decir qué es
import os

nombre = "material"  # cambia este valor para probar
if os.path.exists(nombre):
    if os.path.isdir(nombre):
        print(nombre, "es una CARPETA")
    elif os.path.isfile(nombre):
        print(nombre, "es un ARCHIVO")
else:
    print(nombre, "no existe")

    Ejercicio

    Pide al usuario una ruta y di si existe. Si existe, indica si es archivo o carpeta.

    Solución 
    import os

ruta = input("Introduce una ruta: ").strip()
if os.path.exists(ruta):
    if os.path.isdir(ruta):
        print("Es carpeta")
    elif os.path.isfile(ruta):
        print("Es archivo")
else:
    print("No existe")

    2.3. Crear carpetas

    • os.mkdir("nombre"): crea una sola carpeta (debe no existir).
    • os.makedirs("a/b/c", exist_ok=True): crea varias a la vez; no falla si ya existen.
    # EJEMPLO: crear una carpeta para "exportaciones" y otra estructura para "reportes"
import os

# Crea "export" si no existe; si existe, lanza error -> prueba con try/except o usa makedirs
try:
    os.mkdir("export")
    print("Creada carpeta 'export'")
except FileExistsError:
    print("'export' ya existía")

# Crear varias carpetas anidadas de una vez
os.makedirs("reportes/2026/trimestre1", exist_ok=True)
print("Estructura 'reportes/2026/trimestre1' lista")

    2.4. Listar el contenido de una carpeta

    • os.listdir("ruta"): devuelve la lista de nombres (archivos y subcarpetas).
    # EJEMPLO: listar lo que hay dentro de "reportes" si existe
import os

carpeta = "reportes"
if os.path.isdir(carpeta):
    for nombre in os.listdir(carpeta):
        print("- ", nombre)
else:
    print("No existe la carpeta:", carpeta)

    2.5. Crear y borrar archivos

    • Crear con open(..., "w") y escribir texto.
    • Borrar con os.remove("archivo") si existe.
    # EJEMPLO: crear un archivo de notas y luego borrarlo de forma segura
import os

# Crear y escribir dentro (el "with" cierra el archivo automáticamente)
with open("notas_asix.txt", "w", encoding="utf-8") as f:
    f.write("Tarea 1: repasar os.listdir\n")

# Borrar solo si realmente existe
if os.path.exists("notas_asix.txt"):
    os.remove("notas_asix.txt")
    print("Archivo 'notas_asix.txt' borrado")

    Ejercicio

    2.6. Borrar carpetas (recuerda: deben estar vacías)

    • os.rmdir("carpeta"): borra una carpeta vacía. Si tiene algo dentro, da error.
    # EJEMPLO: intentar borrar una carpeta y manejar el error si no está vacía
import os

objetivo = "reportes/2026/trimestre1"
if os.path.isdir(objetivo):
    try:
        os.rmdir(objetivo)
        print("Borrada:", objetivo)
    except OSError:
        print("No se puede borrar (no está vacía):", objetivo)
else:
    print("No existe:", objetivo)

    Ejercicio

    Crea la carpeta prueba_vaciar, mete un archivo dentro y comprueba que os.rmdir falla. Luego borra el archivo y vuelve a intentar borrar la carpeta.

    Solución 
    import os

os.makedirs("prueba_vaciar", exist_ok=True)
with open(os.path.join("prueba_vaciar", "tmp.txt"), "w", encoding="utf-8") as f:
    f.write("x")

try:
    os.rmdir("prueba_vaciar")
    print("Borrada a la primera (raro)")
except OSError:
    print("Fallo esperado: carpeta no vacía")

os.remove(os.path.join("prueba_vaciar", "tmp.txt"))
os.rmdir("prueba_vaciar")
print("Ahora sí: carpeta borrada")

    2.7. Renombrar o mover archivos

    • os.rename(origen, destino): cambia el nombre o mueve a otra carpeta.
    # EJEMPLO: crear un archivo de demo y renombrarlo
import os

# Crear un archivo de ejemplo
with open("informe.txt", "w", encoding="utf-8") as f:
    f.write("contenido de prueba")

# Renombrar el archivo (mismo sitio)
os.rename("informe.txt", "informe_final.txt")
print("Renombrado a 'informe_final.txt'")

    2.8. Leer una variable de entorno (básico)

    • os.getenv("NOMBRE", "por_defecto"): lee una variable del sistema; si no existe, usa el valor por defecto.
    # EJEMPLO: leer una variable de entorno común y usar un valor por defecto si no está
import os

# En Linux/macOS suele existir USER; en Windows USERNAME
usuario = os.getenv("USER") or os.getenv("USERNAME") or "desconocido"
print("Usuario detectado:", usuario)

    3. Módulo sys: intérprete, argumentos y E/S estándar

    El módulo sys expone funciones y objetos que permiten interactuar con el intérprete de Python: leer argumentos de línea de comandos (sys.argv), trabajar con los flujos estándar (sys.stdin, sys.stdout, sys.stderr), consultar metainformación del entorno (sys.version, sys.platform, sys.executable) y terminar el programa con códigos de salida (sys.exit()). También ofrece utilidades para el control del runtime (recursión, codificación, rutas de importación).

    3.1. Argumentos de línea de comandos: sys.argv

    sys.argv es una lista de cadenas con el nombre del script en argv[0] y los argumentos proporcionados por el usuario a partir de argv[1]. Es útil para utilidades sencillas; para CLIs robustas, se recomienda argparse (validación, tipos, ayuda automática).

    Ejemplo:

    # Lectura de argumentos desde sys.argv
import sys
print("Primer argumento (nombre del script):", sys.argv[0]) # Devuelve el nombre del script
print("Segundo argumento (argumento recibido):", sys.argv[1])  # A partir de aquí, los argumentos del usuario

    Ejercicio

    Crea un código que reciba dos números enteros como argumentos de línea de comandos, los sume e imprima el resultado. Si no se proporcionan dos números, muestra un mensaje de uso y termina con código de error.

    Ver solución 
    # ---------------------------------------------
# Solución: Sumar dos números recibidos por línea de comandos
# ---------------------------------------------
import sys

# Comprobamos que hay, al menos, dos argumentos (además del nombre del script).
# sys.argv[0] -> nombre del script
# sys.argv[1] -> primer número (como str)
# sys.argv[2] -> segundo número (como str)
if len(sys.argv) < 3:
    # Enviamos el mensaje de uso a stderr para diferenciarlo de la salida normal.
    print("Argumentos insuficientes. Se necesitan dos parámetros <num1> <num2>", file=sys.stderr)
    # Terminamos con código 1 para indicar error en los argumentos.
    sys.exit(1)

# Extraemos los argumentos como cadenas.
n1_str = sys.argv[1]
n2_str = sys.argv[2]

try:
    # Convertimos las cadenas a enteros (podría fallar si no son números).
    n1 = int(n1_str)
    n2 = int(n2_str)
except ValueError:
    # Informamos del error por stderr y terminamos con código distinto de 0.
    print("Error: ambos argumentos deben ser enteros.", file=sys.stderr)
    sys.exit(1)

# Calculamos la suma y la mostramos por stdout (salida estándar).
resultado = n1 + n2
print(f"La suma de {n1} + {n2} es: {resultado}")

# En este punto el programa termina de forma natural con código 0 (éxito).

    3.2. Información del intérprete y del entorno de ejecución

    sys ofrece datos útiles para diagnósticos y portabilidad: versión de Python, plataforma del sistema, ejecutable usado, rutas de búsqueda de módulos y codificación del sistema de archivos.

    En el siguiente ejemplo vemos cómo obtener información del entorno:

    
import sys

# Versión completa de Python como cadena; version_info da tupla detallada (mayor, menor, micro).
print("Versión de Python:", sys.version)
print("Version info:", sys.version_info)  # (major, minor, micro, releaselevel, serial)

# Plataforma reportada (ej. 'win32', 'linux', 'darwin').
print("Plataforma:", sys.platform)

# Ruta del ejecutable de Python en uso (útil para depurar entornos con varios intérpretes).
print("Intérprete:", sys.executable)

# Rutas donde Python busca módulos; el orden es importante para las importaciones.
print("sys.path (rutas de importación):")
for p in sys.path:
    print(" -", p)

# Codificación del sistema de archivos (influye en cómo se mapean nombres de ficheros ↔ cadenas).
print("Filesystem encoding:", sys.getfilesystemencoding())

    Ejercicio

    Muestra solo la versión “mayor.menor” (por ejemplo, 3.12) usando sys.version_info, y comprueba si estás en Windows (win32) o Linux (linux) para imprimir un mensaje apropiado.

    Solución 
    import sys
version_mayor_menor = f"{sys.version_info.major}.{sys.version_info.minor}"
print("Versión mayor.menor de Python:", version_mayor_menor)
if sys.platform == "win32":
    print("Estás en Windows")
elif sys.platform == "linux": 
    print("Estás en Linux")
else:
    print("Estás en otra plataforma:", sys.platform)

    3.3. Finalizar el programa con códigos de salida: sys.exit()

    sys.exit(código) termina el programa devolviendo un código al sistema: 0 suele significar éxito; cualquier otro valor indica error. Internamente lanza SystemExit, por lo que puede capturarse si se desea realizar limpieza adicional. 

    # ---------------------------------------------
# Ejemplo 3.3: usar sys.exit para terminar con códigos específicos     
import sys
print("0: Saliendo con éxito  (sys.exit(0))")
sys.exit(0)
print("1: Saliendo con error  (sys.exit(1))")
sys.exit(1)
    Nota: En scripts largos, captura SystemExit solo si necesitas liberar recursos o registrar el evento; de lo contrario, deja que el programa termine naturalmente.

    Ejercicio

    Escribe un programa que pida una ruta por entrada interactiva y verifique si existe. Comprueba mediante el módulo os (del tema anterior) si la ruta existe o no. Si no existe, muestra un mensaje de error y termina con código 1. Si existe, muestra un mensaje de confirmación y termina con código 0.

    Solución 
    # ---------------------------------------------
# Ejemplo 3.3: validar la existencia de una ruta y terminar con códigos apropiados
# ---------------------------------------------
import sys
import os

# Pedimos una ruta por entrada interactiva (podrías usar sys.argv en un caso real).
ruta = input("Introduce una ruta a comprobar: ").strip()

# Si la ruta no existe, informamos por stderr y salimos con código 1 (error).
if not os.path.exists(ruta):
    print(f"Error: la ruta '{ruta}' no existe.", file=sys.stderr)
    sys.exit(1)

# Si existe, confirmamos por stdout y salimos con código 0 (éxito).
print(f"La ruta '{ruta}' existe.")
sys.exit(0)
