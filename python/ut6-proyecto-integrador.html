Introducción General

El proyecto integrador busca consolidar los conocimientos adquiridos en las unidades anteriores mediante el desarrollo de una solución completa que incluya diseño, implementación, validación y mejora continua. Este proyecto simula un caso real y prepara al alumnado para entornos profesionales.

Fase 1: Diseño

1.1 Análisis de necesidades

Todo proyecto parte de una necesidad o problema. Debes responder:

  • ¿Qué funcionalidad principal tendrá la aplicación?
  • ¿Qué tipo de usuario la utilizará?
  • ¿Qué entradas y salidas son necesarias?
  • ¿Qué problema concreto resuelve?

Incluye requisitos funcionales (qué hace el sistema) y no funcionales (rendimiento, seguridad, compatibilidad). Añade una descripción narrativa del proyecto.

Ejemplo: Gestión de biblioteca escolar: registro de libros, préstamos, devoluciones y usuarios.

1.2 Planificación del proyecto

Define fases, tiempos, secuencia, recursos y gestión de versiones. Usa diagramas de Gantt para visualizar tareas.

1.3 Estructura modular del código

Organiza el proyecto en módulos independientes:

Fase 2: Desarrollo

2.1 Implementación de funciones y clases

Aplica PEP8, modularidad y documentación clara.

PEP 8 es la guía oficial de estilo para escribir código Python, que establece convenciones para que el código sea más legible y coherente. Se centra en aspectos como la indentación (usar 4 espacios en lugar de tabuladores), la longitud máxima de línea (79 caracteres), el uso de espacios en blanco, la forma de nombrar variables (usando snake_case) y funciones (snake_case) y el formato de comentarios y docstrings

Reglas clave de PEP 8

  • Indentación: Usa 4 espacios por cada nivel de indentación.

  • Longitud de línea: Limita las líneas a un máximo de 79 caracteres.

  • Líneas en blanco: Usa dos líneas en blanco para separar funciones y clases de nivel superior, y una línea para separar métodos dentro de una clase.

Líneas en blanco: Usa dos líneas en blanco para separar funciones y clases de nivel superior, y una línea para separar métodos dentro de una clase.

Nombres: Usa snake_case (minúsculas separadas por guiones bajos) para variables y funciones, y CamelCase para nombres de clases.

Espacios en blanco: Usa espacios alrededor de los operadores (como =) y después de las comas, pero no dentro de los paréntesis.

Importaciones: Deben estar al principio del archivo, cada una en su propia línea y agrupadas en el orden: biblioteca estándar, bibliotecas de terceros y bibliotecas locales.

Comentarios: Deben ser frases completas que empiecen con mayúscula y expliquen el código, no lo obvio. Las docstrings para funciones, clases y métodos públicos se rigen por el PEP 257. 


class AnalizadorDatos:

        self.ruta_csv = ruta_csv

    def cargar_datos(self):
        import pandas as pd

Ejemplo completo: Gestión de un colegio

Este proyecto integrador consiste en desarrollar una aplicación para gestionar un colegio. Incluye:

  • Registro de alumnos y profesores.
  • Gestión de asignaturas y matrículas.
  • Control de notas y asistencia.

Estructura del proyecto


gestion_colegio/
├── data/
│   ├── alumnos.csv
│   ├── profesores.csv
├── src/
│   ├── main.py
│   ├── utils.py
│   ├── models/
│   │   ├── alumno.py
│   │   ├── profesor.py
│   │   └── asignatura.py
└── docs/

Código base


# models/alumno.py
class Alumno:
    def __init__(self, nombre, edad, curso):
        self.nombre = nombre
        self.edad = edad
        self.curso = curso

# models/profesor.py
class Profesor:
    def __init__(self, nombre, asignatura):
        self.nombre = nombre
        self.asignatura = asignatura

# main.py
from models.alumno import Alumno
from models.profesor import Profesor

alumnos = [Alumno("Ana", 14, "2º ESO"), Alumno("Luis", 15, "3º ESO")]
profesores = [Profesor("María", "Matemáticas"), Profesor("Juan", "Historia")]

for alumno in alumnos:
    print(f"Alumno: {alumno.nombre}, Curso: {alumno.curso}")
